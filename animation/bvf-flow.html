📝 Writing a Behavior Spec
#decl feature auth-feature
Authentication — the gateway to everything.
Get this wrong and nothing else matters.
#behavior login-validates-credentials
Given a registered user with valid credentials.
When they submit the login form.
Then a session is created and they see the dashboard.
#end
#behavior login-rejects-bad-password
Given a registered user with an incorrect password.
When they submit the login form.
Then authentication fails with a generic error.
#end
#end
🔍 Soundness Review — Is the spec clear?
✓
login-validates-credentials
Clear Given/When/Then, testable assertions
?
login-rejects-bad-password
"generic error" — what error? HTTP 401? Flash message? Needs elaboration.
✓
Feature structure
Dependencies resolved, no circular refs
→
Action: mark needs-elaboration
bvf mark login-rejects-bad-password needs-elaboration --note "clarify error response"
⚙️ Materializing Tests
it('login-validates-credentials', async () => {
// Given: registered user with valid creds
const user = await createUser({
email: 'test@example.com',
password: 'correct-horse-battery'
});
// When: submit login form
const result = await login(user.email, user.password);
// Then: session created, sees dashboard
expect(result.session).toBeDefined();
expect(result.redirect).toBe('/dashboard');
});
🎯 Alignment Review — Do tests match spec?
✓
Preconditions
Real user created, not mocked — matches "registered user"
✓
Assertions
Session + redirect verified — covers both "Then" clauses
✗
Scope
Test also checks response time — over-assertion, not in spec
→
Action: mark review-failed
bvf mark login-validates-credentials review-failed --note "over-assertion: response time"
